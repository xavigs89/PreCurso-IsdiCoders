APUNTES! https://www.codecademy.com/learn/learn-html/modules/learn-semantic-html/cheatsheet El HTML semántico se refiere a la práctica de utilizar etiquetas HTML de manera que el código refleje la estructura y el significado del contenido, en lugar de simplemente su apariencia visual. En otras palabras, se trata de escribir HTML de una manera que sea más descriptiva y significativa, facilitando la comprensión tanto para los desarrolladores como para los navegadores y otros agentes que procesan la información. La introducción de etiquetas semánticas en HTML5 ha mejorado significativamente la capacidad de los desarrolladores para estructurar sus documentos web de manera más significativa. Algunas de estas etiquetas semánticas incluyen: 1.
: Representa la cabecera de una sección o de un sitio web. Generalmente contiene elementos como logotipos, títulos y enlaces de navegación. 2.
: Se utiliza para definir una sección de navegación. Contiene generalmente enlaces a otras partes del sitio. 3.
: Indica el contenido principal de la página, excluyendo las cabeceras, pies de página y barras laterales. 4.
: Representa un contenido independiente y autocontenible, como una publicación de blog o un artículo de noticias. 5.
: Define una sección genérica de un documento. Puede contener encabezados, párrafos, listas, etc. 6.
: Se utiliza para marcar contenido que está relacionado con el contenido que lo rodea, pero que puede considerarse independiente. 7.
: Representa el pie de una sección o de un sitio web. Generalmente contiene información de copyright, enlaces de contacto y otros detalles relacionados con el pie de página. El uso de etiquetas semánticas no solo mejora la estructura del código, sino que también tiene beneficios en términos de accesibilidad, SEO y mantenimiento del sitio. Los navegadores y otras herramientas pueden interpretar mejor la estructura del documento, facilitando la indexación de los motores de búsqueda y la navegación para usuarios con discapacidades.
